Send Tab

MWC-QT-Wallet supports several ways to send funds. They fall into two categories:

  1. Address based sending.
  2. File based sending.

There are several types of addresses that are supported.

  1. mwcmqs://<address> - This is MWC Message Queue. This is the main way to send funds in the QT wallet. We are using a fork of Grinbox to allow for users to quickly and easily send funds. The addresses may appear similar to Bitcoin addresses, but the main difference is that in order to send/receive both wallets must be online at the time the funds are sent. This is the default for the wallet.
  2. keybase://<user_name> - Keybase is a tool that allows people to manage keys. It also has an instant messenger functionality. If you install keybase on the system that is running your MWC-QT-Wallet you will be able to send via Keybase. To use this, specify keybase://<user_name> to send to a recipient.
  3. http(s)://<hos>:<port> - You can send to HTTP/HTTPS address. This requires that the host/ip specified be running the MWC full node wallet listener.

File based sending is an interactive 3 step process where the sender initiates a transaction, sends a file (by whatever means) to the recipient. The receiver receives the file which generates a final file which is sent back to the sender. The sender then finalizes the transaction. While this is the most advanced method of sending, it is also the most private.
