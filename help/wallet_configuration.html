Wallet Configuration

This page lets you set key parameters for your wallet. The available options are:

Active features - Select the sending methods you want to use.
- Exchanges typically use Slatepacks.
- To send MWC directly to peers online, you may use Tor or MWCMQS.
Note: Online methods require both the sender and receiver wallets to be online.

Number of confirmations - The number of confirmations required before funds are spendable.
- Default: 10 confirmations (about 1 minute each, compared to Bitcoin's 10 minutes).
- You can reduce this to 1 confirmation if desired, but more confirmations increase security.

Change outputs - The number of change outputs created in a transaction.
- Normally, 1 change output is sufficient.
- Developers may specify more for debugging.
Note: More outputs result in higher transaction fees.

Wallet instance name - The name of this wallet instance. Specify any name that you like.

Wallet data path - Data location for your wallet. You can back up this data, so your transaction history will be safe.

Enable notification windows - Turn on popup notifications in the bottom-right corner of the wallet.

Lock outputs at finalization - Choose when funds are locked during a send:
- At initiation (default), or
- At finalization, keeping funds available until the transaction is finalized.
Locked funds can't be used for another send transaction. To unlock the funds, you can cancel the transaction.

Auto-Logout - Automatically locks the wallet after a set period of inactivity.

Logs - Diagnostic logs are stored at ~/mwc-qt-wallet/logs/mwcwallet.log
- Useful for troubleshooting.
- May contain private information, so disable logging if privacy is more important than debugging.
